В лабораторной работе реализована система многоклассовой классификации изображений собак по породам. В качестве датасета используется **Stanford Dogs Dataset**, архитектура модели — **RegNet-Y-400MF**, основная аугментация по варианту — **RandomRotation**, основной оптимизатор по варианту — **RAdam**. Дополнительно проведено сравнение с оптимизатором Adam и с обучением модели с нуля.

## Ход работы

В ходе лабораторной работы был реализован полный цикл эксперимента по классификации изображений:

1. **Подготовка данных**: загрузка Stanford Dogs Dataset и чтение изображений через `torchvision.datasets.ImageFolder`.
2. **Разбиение выборки**: стратифицированное деление данных на train/val/test в пропорции **70/15/15**.
3. **Предобработка изображений**: изменение размера, центральное кадрирование, преобразование в тензор и нормализация.
4. **Аугментация**: для обучающей выборки применялся случайный поворот изображения `RandomRotation`.
5. **Построение модели**: использована архитектура `regnet_y_400mf` из `torchvision.models`.
6. **Обучение**: проведены эксперименты с предобученной моделью и моделью, обучаемой с нуля.
7. **Оценка качества**: рассчитаны Accuracy, Precision, Recall, F1-score и Loss на тестовой выборке.
8. **Визуализация результатов**: построены графики динамики качества по эпохам для трех экспериментов.

## Теория

### Задача классификации изображений

Классификация изображений — это задача компьютерного зрения, в которой модель получает изображение и должна отнести его к одному из заранее известных классов. В данной работе классы соответствуют породам собак из Stanford Dogs Dataset. Так как классов много, задача относится к **многоклассовой классификации**.

Для обучения используется функция потерь `CrossEntropyLoss`. Она объединяет softmax-преобразование выходов модели и отрицательное логарифмическое правдоподобие правильного класса. Чем выше вероятность правильного класса, тем меньше значение функции потерь.

### RegNet

**RegNet** — семейство сверточных нейронных сетей, построенное как регулярное пространство архитектур. В отличие от ручного проектирования отдельных моделей, RegNet задает простое правило изменения ширины блоков сети. Такой подход делает архитектуру масштабируемой и удобной для практических задач.

В работе используется компактная модель **RegNet-Y-400MF**. Последний классификационный слой заменяется на новый `Linear`, число выходов которого равно количеству пород собак в датасете.

```python
weights = models.RegNet_Y_400MF_Weights.DEFAULT if pretrained else None
model = models.regnet_y_400mf(weights=weights)
model.fc = nn.Linear(model.fc.in_features, num_classes)
```

### Transfer learning

**Transfer learning** — перенос обучения с большой исходной задачи на новую задачу. В данной работе предобученная RegNet уже содержит признаки, полученные на крупном датасете ImageNet: контуры, текстуры, формы и части объектов. Поэтому для Stanford Dogs Dataset модель не учится «с нуля», а дообучается под новую классификационную задачу.

Сравнение с моделью, обученной с нуля, показывает, что transfer learning значительно ускоряет обучение и повышает качество на ограниченном числе эпох.

### Аугментация Rotation

По варианту лабораторной работы используется аугментация **Rotation**. Она случайно поворачивает изображения в обучающей выборке и помогает модели стать устойчивее к наклонам объекта на фотографии.

```python
train_tf = transforms.Compose([
    transforms.Resize(resize_size),
    transforms.RandomRotation(args.rotation),
    transforms.CenterCrop(args.image_size),
    transforms.ToTensor(),
    transforms.Normalize(MEAN, STD),
])
```

### Adam и RAdam

**Adam** — адаптивный оптимизатор, который использует скользящие средние градиента и квадрата градиента. Он часто применяется в задачах глубокого обучения благодаря устойчивой сходимости.

**RAdam** — модификация Adam, в которой исправляется проблема нестабильной дисперсии адаптивного шага на ранних итерациях. За счет корректирующего множителя RAdam может вести себя стабильнее в начале обучения.

### Метрики качества

Для оценки качества использованы следующие метрики:

- **Accuracy** — доля правильных предсказаний среди всех объектов.
- **Precision** — доля корректных объектов среди всех объектов, отнесенных моделью к данному классу.
- **Recall** — доля найденных объектов класса среди всех реальных объектов этого класса.
- **F1-score** — гармоническое среднее Precision и Recall.
- **Loss** — значение функции потерь на тестовой выборке.

Precision, Recall и F1 рассчитаны в **macro-усреднении**: метрика сначала считается отдельно для каждого класса, затем усредняется по всем классам. Такой подход важен для многоклассовой классификации, потому что учитывает качество по каждому классу, а не только общую долю правильных ответов.

## Результаты

Итоговые метрики на тестовой выборке:

| Эксперимент | Предобучение | Оптимизатор | Accuracy | Precision | Recall | F1 | Loss |
|---|---:|---|---:|---:|---:|---:|---:|
| Предобученная RegNet + Adam | да | Adam | **0.7876** | **0.7957** | **0.7842** | **0.7818** | **0.6979** |
| Предобученная RegNet + RAdam | да | RAdam | 0.7771 | 0.7860 | 0.7717 | 0.7710 | 0.7199 |
| RegNet с нуля + Adam | нет | Adam | 0.0656 | 0.0556 | 0.0610 | 0.0432 | 4.3173 |

Лучшее качество показала **предобученная RegNet + Adam**:

- Accuracy: **0.7876**
- Precision: **0.7957**
- Recall: **0.7842**
- F1-score: **0.7818**
- Loss: **0.6979**

Предобученная RegNet с RAdam дала близкое качество: F1-score составил **0.7710**. Модель, обученная с нуля, показала значительно более слабый результат: F1-score равен **0.0432**.

## Анализ результатов

Предобученная RegNet с Adam оказалась лучшей по всем основным метрикам. Разница между Adam и RAdam при дообучении небольшая: Adam выше по F1 примерно на **0.0108**, а по Accuracy — примерно на **0.0105**.

Главный вывод связан не только с выбором оптимизатора, но и с использованием предобучения. Обе предобученные модели значительно превосходят модель, обученную с нуля. Разница между лучшей предобученной моделью и scratch-моделью по F1 составляет **0.7386**.

Это подтверждает, что для задач компьютерного зрения при ограниченном числе эпох transfer learning является более эффективным подходом, чем обучение сверточной сети с нуля.

## Воспроизведение

Примерный порядок запуска эксперимента:

```bash
pip install torch torchvision scikit-learn matplotlib pandas pillow tqdm
```

Пример запуска обучения:

```bash
python src/train.py \
  --data-dir data/StanfordDogs/Images \
  --epochs 6 \
  --image-size 224 \
  --batch-size 32 \
  --lr 1e-4 \
  --weight-decay 1e-4 \

  --optimizer radam \
  --pretrained
```

Для сравнения можно запускать отдельные эксперименты:

```bash
# Предобученная RegNet + Adam
python src/train.py --optimizer adam --pretrained --rotation 20

# Предобученная RegNet + RAdam
python src/train.py --optimizer radam --pretrained --rotation 20

python src/train.py --optimizer adam --rotation 20
```

## Выводы

В лабораторной работе реализована система классификации пород собак на Stanford Dogs Dataset с использованием архитектуры RegNet. Требования варианта выполнены: применена архитектура **RegNet**, аугментация **Rotation** и оптимизатор **RAdam**.

Также проведено дополнительное сравнение с Adam и обучением с нуля. Лучший результат показала предобученная RegNet с Adam: Accuracy = **0.7876**, Precision = **0.7957**, Recall = **0.7842**, F1 = **0.7818**.

Предобученная RegNet с RAdam показала близкий результат: Accuracy = **0.7771**, F1 = **0.7710**. В данном запуске Adam оказался немного лучше RAdam.

Обучение RegNet с нуля за 6 эпох оказалось неэффективным: F1 = **0.0432**. Это подтверждает практическую пользу transfer learning для задач классификации изображений.

Для улучшения качества в дальнейшем можно увеличить число эпох, подобрать learning rate, добавить scheduler, применить более сильные аугментации и провести обучение на более мощном GPU.
